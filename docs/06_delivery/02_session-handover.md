## 🚨 2. Главная техническая проблема: Огромное время сборки (32 минуты)

Сборка образа в GitHub Actions заняла **32 минуты** (28 минут скачивание зависимостей и приложений + 4 минуты компиляция ассетов). Это происходит из-за полной инвалидации кэша Docker-слоев:
* Мы добавили установку `nginx` в начало `Dockerfile` (в команду `RUN apt-get install`). Это привело к сбросу кэша для всех последующих шагов.
* Docker был вынужден заново скачивать все 11 приложений (`bench get-app`) и компилировать фронтенд (`bench build`).

### 💡 Как оптимизировать время сборки (Рекомендации для новой сессии):
1. **Многоэтапный билд или Кэширование:** Настроить GitHub Actions для кэширования папок `~/.cache/pip` и `/home/frappe/.cache/yarn` (или аналогичных директорий внутри контейнера сборщика).
2. **Создание кастомного базового образа (Base Image):** 
   - Собрать отдельный образ `mides-erpnext-base`, в котором уже предустановлены системные утилиты (Supervisor, Nginx) и все стабильные сторонние приложения (HRMS, Insights, Print Designer и т.д.).
   - В основном `Dockerfile` наследоваться от этого базового образа и копировать только меняющиеся файлы конфигурации (`supervisord.conf`, `nginx.conf`, `entrypoint.sh`). Это сократит время сборки до 1–2 минут!
