## 1. Бизнес-контекст

### Компания МиДЭС
- **Тип:** SMB генподрядчик (Казахстан)
- **Структура:** Группа юрлиц — 1 ТОО + 3-5 ИП
- **Масштаб:** 15-30 сотрудников, 3-7 активных строительных проектов
- **Локализация:** Русский язык, KZT (тенге), DD.MM.YYYY, BIN/IIN

### Проблема
Строительная компания работает преимущественно аналоговыми методами:
- Прорабы платят наличными, данные фиксируются в WhatsApp
- Директор не видит полную картину расходов по проектам
- Нет контроля субподрядчиков — оплата без подтверждения выполнения
- Группа юрлиц (ТОО + ИП) усложняет учёт — внутригрупповые переводы ≠ расход
- Бухучёт ведётся в 1С, операционные данные — нигде

### Предыдущая попытка (MidesCloud v3)
Самостоятельная разработка на Next.js 14 + Supabase:
- ✅ Достигнуто: зрелая архитектура (49 ADR), Фаза 0 (Foundation) завершена
- ❌ Причина остановки: чрезмерная сложность собственной разработки для одного разработчика
- 📊 Готовность: 9.5/10 по agent-first readiness, но функционально только 30% от нужного

### Решение
Перенос на ERPNext — зрелую ERP-платформу с 80% нужной функциональности из коробки.

## 3. Ключевые бизнес-процессы

### L1: Bid-to-Cash (Тендер → Деньги)
```
Тендер → Контракт → Строительство → Биллинг → Оплата → Закрытие
```

### L2: Вложенные конвейеры
1. **L2-PRE:** Предстроительный (тендер → договор с заказчиком)
2. **L2-CON:** Контрактование субподрядчиков
3. **L2-A:** Снабжение участка (заявка → закупка → доставка)
4. **L2-B:** Субподрядные работы (заявка → утверждение → оплата → АВР → приёмка)
5. **L2-FIELD:** Полевая отчётность и прогресс
6. **L2-HR:** Табелирование и ЗП
7. **L2-D:** Биллинг заказчика (АВР → ВОР → КС-2/КС-3)
8. **L2-ACC:** Бухгалтерский конвейер (выгрузка в 1С)
9. **L2-FIN:** Финансовый контроль (Budget vs Actual)

### Ретроспективные операции
50-60% расходов — ретроспективная регистрация (прораб уже потратил наличные). Система должна поддерживать обратную запись расходов.

## 4. Критичные бизнес-требования

### Обязательные (Must Have)
1. **Multi-Entity** — управление ТОО + ИП в одной системе
2. **P2P-конвейер** — заявка → утверждение → закупка → оплата
3. **Субподряд** — договор → АВР → физприёмка → оплата → гарантийное удержание
4. **Лицевой счёт подрядчика** — баланс: авансы, АВР, платежи, удержания
5. **Роли и доступ** — директор видит всё, остальные — по назначениям
6. **Русский UI + KZT** — полная локализация
7. **Коды затрат** — 67 позиций по строительным разделам

### Важные (Should Have)
8. **КС-2/КС-3** — формирование актов для заказчика
9. **OCR документов** — распознавание счетов и АВР
10. **Матрица утверждений** — пороги по суммам
11. **ВОР** — ведомость объёмов работ
12. **Budget vs Actual** — контроль бюджета проекта

### Желательные (Nice to Have)
13. **Daily Logs** — ежедневные отчёты с площадки
14. **Интеграция с 1С** — выгрузка бухгалтерских данных
15. **Мобильное приложение** — для прорабов на площадке
16. **AI/OCR** — Gemini для распознавания документов

## 5. Уникальные особенности для Казахстана

- **BIN/IIN** — идентификация юрлиц (12 цифр)
- **КС-2/КС-3/Р1** — нативные формы (западные ERP не поддерживают)
- **ТОО + ИП** — специфика группы юрлиц
- **Гарантийные удержания** — процент от каждого АВР
- **Терпимость ±30-50%** — неточные проекты без формальных Change Orders
- **Аналоговая стройка** — ретроспективная регистрация расходов
