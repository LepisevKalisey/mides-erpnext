# Решение: Миграция на ERPNext

**Статус:** ACCEPTED  
**Дата:** 2026-05-26  
**Тип:** Architecture Decision Record (ADR-01)

---

## Контекст

MidesCloud v3 — строительная ERP-система, разработанная с нуля на стеке Next.js 14 + Supabase + Drizzle ORM. За период разработки:
- Написано 49 ADR, 64 документа, 36 тестов
- Достигнута оценка 9.5/10 по agent-first readiness
- Полностью завершена Фаза 0 (Foundation): auth, юрлица, проекты, контрагенты, сотрудники, договоры
- Частично завершена Фаза 1 (Subcontract): заявки, утверждения, оплата

Однако при масштабировании выявлены проблемы:
1. **Объём работы** — реализация полного ERP-функционала (бухучёт, склад, HR, отчёты) требует 12-18 месяцев разработки
2. **Сложность** — каждый модуль (P2P, AP, AR, GL) требует глубокой экспертизы в предметной области
3. **Поддержка** — один разработчик не может поддерживать production-grade ERP
4. **Бухгалтерия** — пересоздание Chart of Accounts, двойной записи, налогового учёта = пустая трата ресурсов

## Решение

Перенести проект на ERPNext (Frappe Framework), сохранив:
- Бизнес-логику и доменные знания из 49 ADR
- Строительную специфику Казахстана в виде кастомного Frappe-приложения `mides_construction`
- Архитектурные принципы (конвейерная архитектура, Task-per-Role pattern)

## Альтернативы рассмотрены

### 1. Продолжить собственную разработку (MidesCloud v3)
- ✅ Полный контроль, идеальная архитектура
- ❌ 12-18 месяцев до MVP, один разработчик, нет бухучёта
- **Отклонено:** неоправданные трудозатраты

### 2. 1С:ERP + доработки
- ✅ Бухгалтерский стандарт в СНГ, бухгалтеры знают
- ❌ Закрытая платформа, дорогие лицензии, сложная кастомизация
- ❌ Слабый web UI, нет мобильного приложения
- **Отклонено:** не подходит для операционного управления стройкой

### 3. Procore / CMiC / Sage 300 CRE
- ✅ Отраслевые решения мирового класса
- ❌ Не работают с казахстанскими формами (КС-2, КС-3, БИН)
- ❌ Стоимость $500-2000/мес для SMB
- **Отклонено:** не подходит для SMB в Казахстане

### 4. ERPNext + Custom App ← ВЫБРАНО
- ✅ Open source, бесплатно, зрелая платформа (15+ лет)
- ✅ 80% нужного функционала из коробки
- ✅ Frappe Framework позволяет создавать Custom DocTypes
- ✅ Python + MariaDB — знакомый стек
- ✅ Multi-company, Workflows, Permissions — готово
- ⚠️ Нет строительной специфики Казахстана → Custom App
- ⚠️ Требует адаптации UI терминологии на русский

## Последствия

### Положительные
1. Время до MVP сокращается с 12-18 до 4-6 месяцев
2. Бухучёт, склад, HR, отчёты — из коробки
3. Сообщество ERPNext = поддержка и обновления
4. Кастомный app = версионируемый, переносимый

### Отрицательные
1. Зависимость от Frappe Framework (обновления, breaking changes)
2. ERPNext UI менее кастомизируемый, чем собственный React
3. Часть архитектурных решений (AccordionGrid, Sheet-формы) не переносится
4. Нужно изучить Frappe Framework

### Нейтральные
1. 49 ADR остаются как база знаний — бизнес-логику можно перенести
2. Старый репозиторий архивируется, но не удаляется
3. Документация сохраняется в `old_docs/`
