# ADR-02: Разрешение блокировок при первичной настройке ERPNext в Coolify

**Статус:** ACCEPTED  
**Дата:** 2026-05-26  

## Контекст
При первичной настройке ERPNext v16.19.1 на сервере под управлением Coolify возникли две критические ошибки:
1. **Падение Redis Queue (`restarting:unknown`)**:
   Контейнер `redis-queue` постоянно перезапускался, из-за чего сервисы `websocket`, `queue-long` и `queue-short` не могли подключиться к очереди и падали с ошибкой разрешения имени (`Temporary failure in name resolution` для хоста `redis-queue:6379`). Это заблокировало выполнение Setup Wizard на этапе создания пользователя.
2. **Отсутствие русского языка в мастере настройки (Setup Wizard)**:
   Несмотря на наличие русской локализации в репозитории Frappe/ERPNext, русский язык отсутствовал в выпадающем списке выбора языка при первом запуске.

## Решение

### 1. Исправление падения Redis Queue
Причиной постоянного перезапуска контейнера `redis-queue` (в отличие от стабильно работающего `redis-cache`) было монтирование именованного тома:
```yaml
    volumes:
      - redis-queue-data:/data
```
В окружении Coolify это вызывало конфликт прав доступа (права записи пользователя `redis` c UID 999 на примонтированную директорию `/data`). 

**Решение:** Удалено монтирование тома для `redis-queue`. Так как очередь Redis используется для временного хранения фоновых задач и не требует строгой персистентности между перезапусками контейнеров в тестовом окружении, Redis теперь успешно запускается и работает в оперативной памяти (in-memory), аналогично `redis-cache`.

### 2. Включение русского языка (локали `ru`)
В Frappe Framework по умолчанию включены только некоторые основные системные языки. Русский язык (`ru` / `Russian`) присутствует в кодовой базе, но по умолчанию отключён в DocType `Language` в базе данных нового сайта.

**Решение:** Временный контейнер `create-site` переконфигурирован для автоматического поиска существующих сайтов и выполнения на них Python-скрипта через `bench execute`:
```python
import frappe
for name in ["Russian", "ru", "ru-RU"]:
    if frappe.db.exists("Language", name):
        frappe.db.set_value("Language", name, "enabled", 1)
frappe.db.commit()
```
Скрипт был закодирован в Base64 для безопасной передачи через YAML-конфигурацию Docker Compose и выполнен в режиме `eval` (через `exec(__import__('base64').b64decode(...))`), что успешно активировало русскую локаль в базе данных.

## Рассмотренные альтернативы

### Для Redis Queue:
* **Настройка прав на хосте (chown/chmod)**: Требует SSH-доступа к хост-системе Coolify и ручного вмешательства, что нарушает автоматизацию деплоя. Отклонено.
* **Использование локальной папки вместо именованного тома**: Также могло вызвать проблемы с правами в зависимости от настроек Docker-демона Coolify. Отклонено в пользу in-memory работы.

### Для локализации:
* **Ручное включение через Desk**: Невозможно выполнить, так как Setup Wizard блокирует доступ к Desk до завершения настройки, а завершить настройку без выбора правильного языка неудобно.
* **Использование REST API**: Требует авторизации (API Key/Secret), которая генерируется только после прохождения Setup Wizard.

## Последствия
* Все сервисы (`redis-queue`, `websocket`, `queue-long`, `queue-short`) перешли в статус `running` и работают стабильно.
* Setup Wizard успешно пройден на русском языке.
* Стек полностью готов к дальнейшей настройке и тестированию сценариев (ТОО/ИП, Закупки, Субподряд, Биллинг).
