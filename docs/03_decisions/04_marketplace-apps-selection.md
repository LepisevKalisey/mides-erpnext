# Решение: Выбор и интеграция приложений маркетплейса (HR, Print Designer, Insights, S3)

**Статус:** ACCEPTED  
**Дата:** 2026-05-27  
**Тип:** Architecture Decision Record (ADR-04)  

---

## Контекст

Для расширения базового функционала ERPNext v16 под специфику компании МиДЭС был проверен маркетплейс Frappe Cloud. По результатам анализа потребностей бизнеса и ограничений инфраструктуры:
1. Выявлена необходимость автоматизации кадрового табелирования и расчета ЗП (L2-HR конвейер).
2. Выявлена необходимость создания казахстанских форм документов (КС-2, КС-3, Р-1) без ручной верстки.
3. Необходима гибкая BI-отчетность для руководства без доработки бэкенда.
4. Локальный диск сервера ограничен по объему, в связи с чем хранение медиа-файлов, чертежей и документов непосредственно на сервере несет риски переполнения диска.

В то же время, такие модули как *Employee Self Service* (ESS) и *Advanced Procurement* были отклонены из-за избыточной на данном этапе сложности и риска перегрузить пользователей.

---

## Решение

Принято решение установить следующий набор расширений экосистемы Frappe/ERPNext:

1. **Frappe HR (`hrms`):**
   - Установка официального приложения для ведения учета сотрудников, табелей (Attendance) и расчета зарплаты.
2. **Print Designer (`print_designer`):**
   - Установка официального визуального конструктора печатных форм для верстки КС-2, КС-3 и счетов на оплату.
3. **Frappe Insights (`insights`):**
   - Установка BI-приложения для сборки дашбордов маржинальности, бюджетов и закупочной аналитики.

### Замена файлового хранилища на S3
Приложение **Frappe Drive отклонено**. Вместо него хранение всех файлов и вложений документов ERPNext будет переведено на облачное S3-совместимое хранилище (например, MinIO, AWS S3 или Яндекс Облако) через внутренний механизм системы.

Будет применен один из двух путей интеграции S3:
* **Вариант А:** Установка проверенного приложения **`frappe_s3_attachment`** (от Zerodha), которое перехватывает загрузки в DocType `File` и перенаправляет их в баккет.
* **Вариант Б (Инфраструктурный):** Монтирование S3-баккета через `s3fs` или `rclone` в системные директории Docker-тома `sites` (`/sites/mides.localhost/public/files` и `/private/files`), что гарантирует прозрачную работу системы без добавления стороннего кода.

---

## Альтернативы рассмотрены

* **Использование Frappe Drive:**
  - *Плюс:* Удобный интерфейс папок прямо в Desk.
  - *Минус:* Файлы пишутся локально на диск. Риск переполнения сервера при хранении тяжелых фотоотчетов со стройплощадок.
  - *Отклонено* в пользу прямого подключения S3 для всех стандартных вложений ERPNext.

* **Установка ESS (Employee Self Service) и Advanced Procurement:**
  - *Отклонено* как переусложняющие систему на ранних фазах внедрения.

---

## Последствия

### Положительные
1. Существенная экономия ресурсов разработки бэкенда (HR, принты, дашборды и S3-интеграция — готовы «из коробки»).
2. Нулевая нагрузка на дисковую систему сервера благодаря выносу вложений в S3.
3. Директор получает интерактивные отчеты (Insights) с первого дня.

### Риски и митигация
* **Сложность деплоя в Docker:** Стандартный Docker-образ `frappe/erpnext:v16` не содержит приложений `hrms`, `print_designer` и `insights`. 
* **Митигация:** Будет подготовлен кастомный Dockerfile для сборки собственного образа ERPNext с предустановленным списком приложений (через команду `bench get-app`). Собранный образ будет задеплоен через Coolify.
