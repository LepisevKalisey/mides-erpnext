# ADR-39: Тип работ (work_stream) на уровне объекта строительства

**Статус:** Принято  
**Дата:** 2026-05-18  
**Авторы:** Backend Agent

---

## Контекст

Строительный проект в MidesCloud состоит из объектов строительства (`project_objects`).
Каждый объект имеет два направления работ:
- **CIVIL** — Общестрой (конструктивные, отделочные, земляные работы)
- **MEP** — Инженерные сети (механика, электрика, сантехника)

Ранее `work_stream` хранился только в `project_assignments`, что не позволяло фильтровать
объекты по типу и было семантически неверно: тип работ — свойство объекта, а не назначения.

## Решение

Добавить колонку `work_stream TEXT CHECK (work_stream IN ('CIVIL', 'MEP')) DEFAULT 'CIVIL' NOT NULL`
прямо в таблицу `project_objects`.

### Последствия для UI

- При добавлении объекта пользователь выбирает тип работ (CIVIL / MEP) через toggle-кнопки
- Список объектов группируется по типу (Общестрой / Инженерные сети)
- При назначении сотрудника выпадающий список объектов показывает тип рядом с именем

### Связь с назначениями

`project_assignments.work_stream` остаётся — он описывает поток работ конкретного сотрудника
в рамках объекта (более детальная разбивка). `project_objects.work_stream` — основной тип.

## Альтернативы

- Хранить только в `project_assignments` — отклонено: нет единого места правды для типа объекта
- Отдельная таблица типов — избыточно для бинарного выбора CIVIL/MEP

## Последствия

- ✅ Чёткое разделение: объект знает свой тип
- ✅ UI может группировать объекты по CIVIL/MEP
- ⚠️ Все существующие записи `project_objects` получают DEFAULT 'CIVIL'
