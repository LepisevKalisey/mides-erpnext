# ADR-04: Структура контрагентов — холдинг и юридические лица

**Статус:** Accepted  
**Дата:** 2026-05-04  
**Автор:** Архитектор системы

---

## Контекст

Один контрагент (компания, группа, холдинг) может вести деятельность через несколько юридических лиц. Договоры подписываются конкретным юрлицом, а не материнской компанией. При этом для отчётности и аналитики важна группировка по головному контрагенту.

Пример:
```
ТОО "СтройГрупп" (холдинг)
├── ТОО "СтройГрупп-Монтаж"  БИН: 123456789012
├── ТОО "СтройГрупп-Снабжение"  БИН: 987654321098
└── ИП Иванов А.А.  ИИН: 111122223333
```

## Решение

Двухуровневая структура контрагентов:

```sql
-- Головной контрагент (холдинг, группа, физлицо-основатель)
contractors (
  id, name, note
)

-- Юридические лица под контрагентом
contractor_entities (
  id,
  contractor_id → contractors(id),
  name,         -- полное юрид. наименование
  bin,          -- БИН/ИИН
  phone,
  legal_form    -- ТОО / АО / ИП / ГП и т.д.
)
```

### Связи в системе

- **Договоры** ссылаются на `contractor_entity_id` (конкретное юрлицо)
- **Заявки** ссылаются на `contractor_id` (головной контрагент, для отчётности)
- **Платежи** ссылаются на `contractor_entity_id` (платим конкретному юрлицу)

### UI

- Справочник контрагентов: аккордеон «Холдинг → список юрлиц»
- Форма создания договора: сначала выбор холдинга, потом юрлица из его списка
- Выбор подрядчика в заявке: выбор холдинга (группировка по нему)

## Последствия

- Миграция: добавить `contractor_entities`, удалить поля `bin`/`phone` из `contractors`
- Обновить внешние ключи в `contracts` (добавить `contractor_entity_id`)
- Обновить форму создания договора (двухшаговый выбор контрагента)

## ADR-05 — Accordion Tables

**Статус:** Accepted  
**Дата:** 2026-05-04

Все табличные представления реализуются как **accordion tables**: каждая строка является раскрываемой. При клике строка разворачивается и показывает детальную информацию, связанные записи или действия.

### Компонент

```tsx
<AccordionTable>
  <AccordionTableRow
    summary={<>...ячейки...</>}
    detail={<>детали/дочерние строки</>}
  />
</AccordionTable>
```

### Применение

- Справочник контрагентов: холдинг → юрлица
- Реестр заявок: заявка → история платежей
- Реестр платежей: платёж → список заявок
- Реестр договоров: договор → связанные заявки/АВР
