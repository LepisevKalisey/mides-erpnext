# ADR-05: Dashboard как «Рабочий день» и назначение менеджера контрагентов

## Статус
Принято — 2026-05-04

## Контекст

При первоначальном проектировании «Рабочий день» был реализован как первая вкладка внутри экрана Казначейства (`/treasury?tab=workday`). Это создаёт две проблемы:

1. Рабочий день логически не принадлежит одному отделу — это агрегированный стартовый экран для **любой роли** (казначей видит очередь платежей, юрист — список договоров на подписание, снабженец — своих контрагентов и т.д.).
2. В системе существует класс сотрудников, которые взаимодействуют с внешними контрагентами (снабженцы, кураторы субподрядчиков). Для них нужен выделенный инструмент управления контрагентами в дашборде.

## Решение

### 1. Dashboard = Рабочий день

- `/dashboard` становится **стартовой страницей** для всех ролей.
- Содержимое рендерится в зависимости от набора ролей текущего пользователя.
- Разделы дашборда добавляются по мере реализации новых отделов.
- `Рабочий день` выносится из `/treasury` — казначейство сохраняет только данные-ориентированные вкладки (реестры, контрагенты и т.п.).

### 2. Флаг `works_with_contractors` на уровне пользователя

- В таблицу `user` добавляется поле `worksWithContractors: boolean (default: false)`.
- Флаг управляется **только администратором** (через `UserRolesSheet` или отдельный экран настроек).
- Если флаг включён, в дашборде пользователя появляется вкладка **Контрагенты**.

### 3. Привязка контрагента к сотруднику-куратору

- В таблицу `contractors` добавляется поле `managerId: text (FK → user.id, nullable)`.
- При создании или редактировании контрагента администратор или куратор выбирает ответственного сотрудника.
- Контрагент без `managerId` — «неназначенный» (видят только администраторы).
- Сотрудник с `worksWithContractors = true` видит в дашборде только **своих** контрагентов (`managerId = currentUserId`).

## Альтернативы

| Подход | Причина отклонения |
|---|---|
| Оставить Рабочий день в `/treasury` | Логика не должна быть привязана к одному отделу |
| Использовать `responsibilities` для привязки контрагентов | `responsibilities` — 3-ключевой junction (user+object+contractor); для куратора нужна прямая связь |
| Отдельная таблица `userSettings` | Избыточно при наличии кастомных полей в `user` |

## Последствия

- При добавлении новой роли (например, `SUPPLY_MANAGER`) нужно добавить секцию в `DashboardPage`.
- Содержимое вкладки «Контрагенты» в дашборде определяется отдельно (следующий ADR или спецификация).
- Скринам казначейства обновить ключи: убрать `finance.treasury.workday` как отдельный экран.
