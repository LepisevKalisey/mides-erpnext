# ADR-06: Двухуровневая структура Cost Codes

**Статус:** Принято  
**Дата:** 2026-05-10

---

## Контекст

В системе существуют два перечня видов работ:

1. **Перечень видов подрядных работ РК** — утверждён законодательством Казахстана (850 строк). Плоская структура, одноуровневый список. В договорах с заказчиком используются именно эти формулировки.

2. **cost_code.csv** — внутренняя операционная таблица (1001 строк) с четырёхуровневой иерархией: Раздел → Категория → Вид работ → Теги. Удобна для работы на площадке, содержит поисковые теги на русском.

**Проблема:** Нельзя выбросить перечень РК — это юридическое требование. Нельзя работать только с перечнем РК — он неудобен для оперативной работы.

---

## Решение: Двухуровневая структура с маппингом

```
УРОВЕНЬ 1: Официальный перечень РК (legal_work_types)
  Источник: Перечень видов подрядных работ (услуг) в составе СМР
  Использование:
    - Договор с заказчиком (ВОР)
    - КС-2 / КС-3 (официальная отчётность)
    - Юридически значимые документы

УРОВЕНЬ 2: Операционные Cost Codes (cost_codes)
  Источник: cost_code.csv (Раздел → Категория → Вид → Теги)
  Использование:
    - Создание заявок INITIATOR
    - Маппинг АВР субчиков
    - RAG-подсказки («введите 'монолит' → покажем все позиции»)
    - Накопительная ведомость ПТО

МАППИНГ (cost_code_mappings):
  Операционный Cost Code ↔ Официальная строка РК (M:M)
  Делает: HEAD_OF_PTO или ADMIN при настройке проекта
  Пример:
    «Устройство монолитных плит перекрытия» (cost_code)
      → «Устройство монолитных конструкций» (legal_work_type)
```

---

## Структура БД

```sql
-- Официальный перечень РК (неизменный, загружается из CSV)
legal_work_types (
  id,
  section_number,        -- "3."
  section_title,         -- "Возведение несущих конструкций"
  item_number,           -- "3.1"
  description,           -- "Устройство монолитных конструкций"
  source                 -- "Постановление РК №XXX"
)

-- Операционные коды (из cost_code.csv)
cost_codes (
  id,
  section,               -- "КЖ"
  category,              -- "Монолитный каркас"
  work_type,             -- "Устройство монолитных плит перекрытия"
  tags,                  -- ["перекрытие", "монолит", "плита", "бетон"]
  is_active
)

-- Маппинг (M:M)
cost_code_mappings (
  cost_code_id,
  legal_work_type_id,
  note,                  -- пояснение к маппингу
  mapped_by,
  mapped_at
)
```

## Где используется каждый уровень

| Экран / функция | Уровень |
|---|---|
| Создание заявки INITIATOR | Операционный (cost_codes) |
| Маппинг АВР → ВОР (ПТО инженер) | Операционный → через маппинг → Официальный |
| ВОР строки (договор с заказчиком) | Официальный (legal_work_types) |
| КС-2 заказчику | Официальный (legal_work_types) |
| Поиск/RAG-подсказки | Теги из cost_codes |
| Накопительная КС-6а (внутренняя) | Операционный (cost_codes) |
| Накопительная для заказчика | Официальный (legal_work_types) |
| Справочник (раздел меню) | Оба (вкладки) |

---

## Фазированная загрузка данных

**Фаза 1 (seed при деплое):**
- Загрузить `cost_code.csv` → таблица `cost_codes`
- Загрузить перечень РК → таблица `legal_work_types`
- Маппинг создаётся вручную HEAD_OF_PTO по мере необходимости

**Фаза 2:**
- Предзаполненный маппинг типовых позиций (80% случаев)
- RAG-поиск по тегам для INITIATOR при создании заявки

---

## Последствия

- Юридическая чистота: КС-2 всегда в терминах перечня РК
- Удобство работы: INITIATOR работает с понятными операционными терминами
- ПТО инженер настраивает маппинг один раз — потом работает автоматически
- Возможность аудита: видно как операционные работы связаны с официальными
