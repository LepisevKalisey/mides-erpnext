# ADR-020: Лимит оборота с контрагентом без договора

**Status:** Accepted  
**Date:** 2026-05-07

## Context

Законодательство РК и внутренняя политика компании: оборот с одним контрагентом без
подписанного договора не должен превышать 1 000 000 ₸ в год.

При превышении лимита бухгалтер должен инициировать заключение договора.

Цель: предотвратить налоговые риски и создать правовую базу для крупных отношений.

## Decision

### Схема данных

```sql
CREATE TABLE contractor_annual_volume (
  contractor_id UUID REFERENCES contractors(id),
  year          INT NOT NULL,
  total_paid    NUMERIC NOT NULL DEFAULT 0,
  threshold     NUMERIC NOT NULL DEFAULT 1000000,
  alert_sent    BOOLEAN NOT NULL DEFAULT false,
  PRIMARY KEY (contractor_id, year)
);
```

### Правила

1. **При каждом подтверждённом платеже** контрагенту без договора:
   ```
   UPDATE contractor_annual_volume
   SET total_paid = total_paid + payment.amount
   WHERE contractor_id = X AND year = CURRENT_YEAR
   ```

2. **Триггер предупреждения** (не блокировки):
   - `total_paid >= threshold` И `contractor.contracts = 0 для этого года`
   - → `alert_sent = true`
   - → Уведомление в `notification_queue` → бухгалтер (роль ACCOUNTING)
   - → Бухгалтер видит в своём разделе: «Контрагенты без договора, превысившие лимит»

3. **Платежи не блокируются** — только предупреждение.

4. **Автоматический сброс**: 1 января каждого года `total_paid = 0`, `alert_sent = false`
   (реализуется через scheduled Edge Function).

5. **Контрагент с договором** — лимит не применяется, трекинг не ведётся.

### Отображение

Страница «Контрагенты» (роль ACCOUNTING):
```
⚠️ Контрагенты без договора, превысившие лимит (2026)
──────────────────────────────────────────────────────
Иванов ИП        1 250 000 ₸ / 1 000 000 ₸  [Инициировать договор]
ТОО «Ресурс»       980 000 ₸ / 1 000 000 ₸  [Следить]
```

## Alternatives Considered

- **Блокировать платежи при превышении** → отклонено, т.к. замедляет операционную работу
- **Требовать подтверждения директора при каждом платеже сверх лимита** → отклонено, избыточно
- **Не отслеживать** → налоговый риск

## Consequences

- **+** Бухгалтер видит картину без ручного мониторинга
- **+** Операционная работа не прерывается
- **+** Лимит настраивается в `procurement_settings` (можно изменить)
- **−** Не защищает от намеренного обхода (несколько платежей мелкими суммами)
