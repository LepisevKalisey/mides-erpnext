# ADR-017: Гарантийное удержание в субподрядных договорах

**Status:** Accepted  
**Date:** 2026-05-07

## Context

Компания работает в двух позициях:

**Как субподрядчик (перед заказчиком):** заказчик удерживает 3–10% из каждого платежа на гарантийный период. Эта сумма — будущая дебиторская задолженность компании.

**Как генподрядчик (перед субподрядчиком):** компания удерживает 3–10% из каждого платежа субподрядчику на гарантийный срок. Эта сумма — кредиторская задолженность перед субподрядчиком (будет возвращена после истечения гарантии).

В строительной практике РК термин — «гарантийное удержание» или «гарантийный процент» (в отличие от retainage в англоязычной практике — суть идентична).

## Decision

### Схема данных

Таблица `contracts`:
```sql
warranty_percent        NUMERIC    -- 3–10%, задаётся при заключении договора
warranty_period_months  INT        -- срок гарантии в месяцах
warranty_start_date     DATE       -- дата начала гарантийного периода
warranty_amount_held    NUMERIC    -- накопленная сумма удержания
warranty_release_date   DATE       -- расчётная дата возврата
warranty_released_at    TIMESTAMPTZ -- фактическая дата возврата (заполняется при выплате)
```

Таблица `requests` (устарела, на практике поля перенесены в `payment_tasks` согласно ADR-38 и ADR-43):
```sql
-- В таблице payment_tasks:
warranty_retention_amount  NUMERIC  -- сумма удержания по этому платежу
net_payment_amount         NUMERIC  -- к выплате = amount_paid - warranty_retention_amount
```

### Расчёт

```
warranty_retention_amount = amount_paid * (contract.warranty_percent / 100)
net_payment_amount = amount_paid - warranty_retention_amount
```

Поле `contracts.warranty_amount_held` накапливается при каждом подтверждённом платеже.

### Отображение в карточке субподрядчика

```
Гарантийное удержание:  500 000 ₸
  Срок гарантии:        24 месяца
  Дата возврата:        01.03.2028
  Статус:               Удерживается / Возвращено
```

### Возврат гарантийного удержания

Возврат оформляется отдельным `request` с типом `WARRANTY_RELEASE`:
- Не требует обычного согласования (только директор)
- Автоматически заполняет `contracts.warranty_released_at`

### Уведомление о возврате

За 30 дней до `warranty_release_date` → уведомление директору:
«Приближается дата возврата гарантийного удержания субподрядчику [X]: [сумма] ₸».

## Alternatives Considered

- Не фиксировать гарантию в системе → потеря контроля над cash flow и обязательствами
- Хранить только в договоре без трекинга по платежам → невозможно знать актуальную сумму удержания

## Consequences

- **+** Точный учёт обязательств перед субподрядчиками
- **+** Корректный cash flow прогноз (warranty_release_date = будущий outflow)
- **+** Соответствие налоговому учёту: удержанная гарантия не является расходом до выплаты
- **−** Требует дисциплины при вводе договора: warranty_percent и срок должны заполняться при создании
