# ADR-028: Стратегия внедрения Material Intelligence и GraphRAG

**Статус:** Принято  
**Дата:** 2026-05-08  
**Связано с:** [ADR-22](22_sourcing-workspace.md), [06_graphrag-readiness.md](../04_architecture/06_graphrag-readiness.md), [05_material-intelligence-design.md](../04_architecture/05_material-intelligence-design.md)

---

## Контекст

Предложено две системы, требующих решения о времени реализации:

1. **Material Intelligence** — граф синонимов материалов, обогащение тегов из маппинга счетов, рекомендации поставщиков
2. **GraphRAG + Gemma 4** — знаниевый граф с LLM-извлечением сущностей, векторным поиском, свободными запросами

**Ключевое ограничение:** GraphRAG бесполезен без данных. Первые 3–6 месяцев работы системы граф слишком разрежен для качественных рекомендаций. Внедрять полный GraphRAG до накопления данных — потратить ресурсы впустую.

**Второе ограничение:** GPU-сервер для Gemma 4 — отдельная инфраструктурная задача, не зависящая от разработки.

---

## Решение: трёхэтапная стратегия

### Этап 1 — Фаза 3 (СЕЙЧАС, в рамках текущего релиза)

**Что делаем:** Только фундамент данных. Без LLM. Без сложности.

**Задачи:**

```sql
-- 1. Включить расширения (1 команда, без рисков)
CREATE EXTENSION IF NOT EXISTS vector;        -- pgvector 0.8.0, уже доступен
CREATE EXTENSION IF NOT EXISTS pg_trgm;       -- нечёткий поиск

-- 2. Новая схема счетов (вместо текущего sourcing_quotes)
CREATE TABLE supplier_invoices (...);
CREATE TABLE invoice_lines (...);             -- с полем embedding vector(768)
CREATE TABLE quote_line_mappings (...);

-- 3. Material Catalog
CREATE TABLE material_catalog (...);          -- с полем embedding vector(768)
CREATE TABLE material_aliases (...);          -- синонимы + confidence

-- 4. Триграм-индекс для нечёткого поиска
CREATE INDEX ON invoice_lines USING gin (description gin_trgm_ops);
CREATE INDEX ON material_catalog USING gin (canonical_name gin_trgm_ops);
```

**Рекомендации в этом этапе:** через `pg_trgm` (ILIKE + similarity). Без LLM.  
Пример: «Эта строка счёта похожа на "Арматура д12" (сходство 87%)».

**Обогащение тегов:** после confirmed маппинга → alias записывается в `material_aliases`.  
Это ручной процесс подтверждения снабженцем, без автоматики.

**Оценка:** +1 день к Фазе 3 (схема расширяется, UI не меняется значительно)

---

### Этап 2 — v1.1 (после завершения всех Фаз 0–10, ~через 1–2 месяца после прода)

**Условие запуска:** система в проде, идут реальные транзакции.  
**Триггер:** накоплено 50+ подтверждённых маппингов.

**Что делаем:** Включаем векторные эмбеддинги и умные рекомендации.

**Задачи:**
- Edge Function `generate-embeddings`: новая `invoice_line` / `material_catalog` запись → вызов **Google text-embedding-004** (бесплатный, качественный русский) → сохранение в `embedding` column
- HNSW-индекс на `embedding` колонках
- Замена pg_trgm рекомендаций на vector similarity (`<=>` оператор)
- Прайс-трекер: `material_price_history` view — средняя цена материала за период
- В сайдбаре снабженца: «Последние 5 закупок: 430–460 ₸/кг. Текущее КП: 480 ₸/кг (+7%)»

**LLM:** Gemini Flash API (без self-hosted сервера, низкая стоимость, хороший русский)  
Только для генерации обоснований / суммаризации поставщика. Данные о ценах — из структурированной БД, не из LLM.

**Оценка:** 3–5 дней разработки

---

### Этап 3 — v2.0 (через 3–6 месяцев после прода)

**Условие запуска:**
- В системе 500+ подтверждённых маппингов (граф не пустой)
- GPU-сервер с Gemma 4 развёрнут и стабилен
- Команда протестировала качество Gemma 4 на реальных данных компании

**Что делаем:** Полный GraphRAG.

**Задачи:**
- `knowledge_nodes` + `knowledge_edges` таблицы
- `pgmq` очередь событий: маппинг подтверждён → event → knowledge-builder
- Edge Function `knowledge-builder`: вызов Gemma 4 → извлечение сущностей → обновление графа
- GraphRAG retrieval функция (vector + graph traversal)
- «Типовая корзина» по видам работ (co-occurrence)
- Аномалии: цена подозрительно низкая / высокая (сравнение с графом)
- (Опционально) Chat-интерфейс: свободные вопросы к базе знаний

**LLM:** Gemma 4 self-hosted (Ollama) для извлечения сущностей. Gemini API — fallback.

**Оценка:** 10–15 дней разработки

---

## Зависимости и решения по LLM

| Этап | Когда | LLM | Почему |
|---|---|---|---|
| 1 (Фаза 3) | Сейчас | Нет | Данных ещё нет |
| 2 (v1.1) | +1–2 мес после прода | Google text-embedding-004 + Gemini Flash | Без сервера, хороший русский |
| 3 (v2.0) | +3–6 мес после прода | Gemma 4 self-hosted | Конфиденциальность + данные накопились |

**Почему Gemini Flash, а не Gemma 4 для v1.1:**  
- GPU-сервер — отдельная задача, не блокирует разработку  
- Gemini Flash дешевле, быстрее, лучше понимает русский строительный язык  
- Данные для embedding — не критично конфиденциальны (canonical names материалов)  
- Переход на self-hosted — в v2.0, когда есть смысл инвестировать в инфраструктуру

---

## Что НЕ делаем

- ❌ GraphRAG в Фазе 3 — нет данных, трата времени
- ❌ Apache AGE в Supabase — расширение недоступно, adjacency tables достаточно
- ❌ Обязательный LLM в Фазе 3 — усложняет MVP без выгоды
- ❌ Ручная предзагрузка material_catalog из КССС — слишком большой проект, растём органически

---

## Альтернативы, которые не выбраны

| Вариант | Почему отклонён |
|---|---|
| GraphRAG с первого дня | Нет данных → пустой граф → плохие рекомендации → недоверие пользователей |
| Купить готовое AI-решение (Coupa AI, Ariba Spot Buy) | Дорого, не интегрируется с нашей схемой, не знает русский контекст |
| Подождать с LLM до v3 | Embeddings в v1.1 — низкий риск, высокая ценность, не требует GPU |

---

## Последствия

**Положительные:**
- Фаза 3 не усложняется — только схема данных, без AI-кода
- Данные накапливаются корректно с первого дня (embeddings-ready schema)
- v1.1 даёт реальную ценность (price tracker, vector recommendations) с минимальными рисками
- v2.0 GraphRAG приходит когда есть и данные, и понимание качества Gemma 4 на нашем домене

**Отрицательные / риски:**
- GPU-сервер для Gemma 4 нужно планировать до v2.0 (3–4 мес на подготовку)
- Качество Gemma 4 на русском строительном языке — неизвестно, нужно тестировать заранее
- Если конкурент запустит AI-рекомендации раньше — мы позади на 3–6 мес (приемлемо для B2B)

## Чеклист для перехода на следующий этап

**v1.1 открывается когда:**
- [ ] Фазы 0–10 завершены и система в проде
- [ ] 50+ подтверждённых маппингов `quote_line_mappings`
- [ ] Google API key настроен в Supabase Vault

**v2.0 открывается когда:**
- [ ] 500+ подтверждённых маппингов
- [ ] GPU-сервер с Ollama + Gemma 4 развёрнут и протестирован
- [ ] Качество entity extraction проверено на 50+ реальных счетах компании
- [ ] knowledge_nodes/edges схема согласована
