## 1. Список ролей

В системе MidesCloud разделяются глобальные постоянные роли сотрудников компании (`system_role`) и временные назначения на конкретные проекты/объекты (`project_role`).

### 1.1 Постоянные роли компании (`system_role`)

| Код роли | Название | Описание и сфера ответственности |
|---|---|---|
| `DIRECTOR` | Генеральный директор | Полный доступ. Утверждение без лимитов. Управление матрицей полномочий. |
| `DEPUTY_DIRECTOR [CIVIL]` | Зам по общестрою | Согласование по общестроительным объектам в рамках лимитов матрицы. |
| `DEPUTY_DIRECTOR [MEP]` | Зам по инженерным сетям | Согласование по инженерным сетям в рамках лимитов матрицы. |
| `DEPUTY_DIRECTOR [GENERAL]` | Зам по общим вопросам | Согласование по общим расходам и накладным расходам. |
| `HEAD_OF_PROCUREMENT` | Начальник снабжения | Распределение пула заявок, контроль SLA, аналитика поставщиков. |
| `BUYER` | Снабженец | Обработка назначенных заявок, проведение тендеров, создание PO. |
| `WAREHOUSE` | Кладовщик | Физический приём материалов на центральный склад, оформление GRN. |
| `HEAD_OF_PTO` | Начальник ПТО | Контроль объёмов выполнения работ, ведение ВОР, распределение задач ПТО. |
| `PTO_ENGINEER` | Инженер ПТО | Проверка АВР (физические объёмы), ведение накопительной. |
| `SITE_SUPERINTENDENT` | Прораб | Формирование заявок на материалы (PR) со строительной площадки. |
| `ECONOMIST` | Экономист | Контроль проектных бюджетов, сквозной финансовый анализ. |
| `HEAD_OF_ACCOUNTANTS` | Главный бухгалтер | Согласование реестров платежей, контроль бухгалтерской отчётности. |
| `ACCOUNTANT` | Бухгалтер | Проверка финансовых реквизитов, фиксация ЭСФ, проведение платежей в 1С. |
| `TREASURER` | Казначей | Управление реестром оплат, совершение фактических платежей. |

### 1.2 Проектные роли (`project_role` через `project_assignments`)

Проектные назначения привязаны к конкретному проекту или объекту строительства и служат для разграничения прав доступа и автоматического роутинга согласований.

| Код роли | Название | Функция в проекте |
|---|---|---|
| `GI` | Главный инженер проекта (ГИП) | Технический контроль. Подтверждает физические объёмы и работы по АВР. |
| `PM` | Руководитель проекта (РП) | Коммерческий контроль. Утверждает суммы АВР, договора и доп. соглашения. |
| `SITE_MANAGER` | Начальник участка | Физическая приёмка работ субподрядчиков с фотофиксацией на объекте. |
| `PTO_ENGINEER` | Инженер ПТО проекта | Операционное маппирование АВР на ведомость объёмов работ (ВОР). |
| `BUYER` | Снабженец проекта | Закреплённый за проектом снабженец для оперативной обработки заявок. |
| `ACCOUNTANT` | Бухгалтер проекта | Контроль первичных документов по конкретному проекту. |
| `WAREHOUSE` | Кладовщик проекта | Приёмка материалов непосредственно на приобъектный склад. |

## 4. Дашборды по ролям

* **DIRECTOR / DEPUTY**: Реестр счетов на утверждение, общие затраты по проектам, KPI маржинальности, отклонения от бюджета.
* **PM (Руководитель проекта)**: Мои проекты, текущие договора, исполнение бюджета, согласование допсоглашений.
* **BUYER (Снабженец)**: Личный пул заявок в работе, создание PO, отслеживание поставок.
* **SITE_SUPERINTENDENT / SITE_MANAGER**: Быстрое создание заявок на материалы, подтверждение приёмки грузов, фотофиксация АВР.
* **TREASURER (Казначей)**: Единый платёжный календарь, выгрузка платёжек в банк-клиент, отметка о факте оплаты.
* **ACCOUNTANT (Бухгалтер)**: Проверка первичных документов (АВР, счета-фактуры), сверка с 1С.
